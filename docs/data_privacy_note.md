# Data Privacy Note

Este projeto usa somente dados sintéticos. Não há dados pessoais reais, dados sensíveis, bases de clientes reais ou informações de empresas reais.

Os IDs de lead, aluno, campanha e criador são fictícios. Eles existem apenas para simular relacionamentos analíticos entre marketing, vendas, produto, CRM e receita.

O projeto não usa APIs externas. Ele foi criado apenas para fins de portfólio, estudo e demonstração de raciocínio analítico em Marketing Analytics, Revenue Analytics e Revenue Governance.

Em produção, uma solução parecida exigiria cuidados de privacidade e LGPD desde o desenho. Isso incluiria consentimento adequado, minimização de dados, controle de acesso, mascaramento de campos sensíveis, políticas de retenção, auditoria de uso, governança de tracking, documentação das fontes e lineage básico.

Em linguagem simples: coletar só o necessário, explicar o uso dos dados, proteger o acesso, remover ou mascarar o que não precisa identificar uma pessoa e manter rastreabilidade sobre origem, uso e descarte dos dados.
